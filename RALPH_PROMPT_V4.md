# Research Report - Ralph Loop Prompt V4

## タスク
{topic}についての**エキスパートレベル**のリサーチレポートをPDFで作成。

## 画像に関する厳格ルール（最重要）

### 禁止事項
- ❌ 抽象的なイメージ画像（カラーパレット、ワークスペース写真など）
- ❌ 「○○ trend」で適当に検索した画像
- ❌ 1トピック1枚の機械的な配置
- ❌ 内容と関係ない装飾画像

### 必須事項
- ✅ 参照記事で実際に使われている図表・スクリーンショット
- ✅ 具体的なデータを示すチャート・グラフ
- ✅ 製品/サービスの実際のUI画像
- ✅ 文脈に沿った場所に配置（本文中に埋め込む）

### 画像取得フロー
1. WebFetchで記事を読む
2. 記事内で引用されている図表のURLを特定
3. Playwrightで実際にページを開きスクリーンショット or 画像URL取得
4. 画像をClaudeで確認し、**具体的で情報価値がある**か判定
5. 抽象的・装飾的なら却下して別を探す

## 体裁ルール

### PDF生成時
- フォント: サンセリフ系（Inter, Helvetica Neue, Arial）
- 本文: 16px以上
- H1: 32px / H2: 24px / H3: 18px
- 行間: 1.9
- 余白: 十分に確保

### レイアウト
- 画像は関連する本文の直後に配置
- キャプションで画像の出典を明記
- 余計な装飾を排除、情報密度を高く

## 実行フロー

### Phase 1: 深掘りリサーチ
1. WebSearch 15回以上（異なるアングルから）
2. 主要ソース5つ以上をWebFetchで精読
3. 各ソースから具体的な図表・データを抽出

### Phase 2: 画像取得（最重要）
1. 参照記事内の有用な図表を特定
2. Playwrightでブラウザ操作、スクリーンショット取得
3. 取得画像をClaudeで評価:
   - 「この画像は具体的なデータ/UIを示しているか？」
   - 「抽象的なイメージではないか？」
   - 評価結果を記録
4. **5枚以上の具体的画像**を確保するまで繰り返す

### Phase 3: レポート執筆
1. TL;DR（冒頭5行以内）
2. 本文（画像を適切な位置に埋め込み）
3. 出典セクション（URL 10個以上）

### Phase 4: PDF生成 & 検証
1. generate_pdf.pyでPDF作成
2. PDFを開いて目視確認:
   - 画像が具体的か？
   - フォントが読みやすいか？
   - 余白は十分か？

## 品質評価基準（厳格）

### 評価A（合格）- 全て満たす必要あり
| 項目 | 基準 |
|------|------|
| WebSearch | 15回以上 |
| 参照ソース | 5サイト以上を深読み |
| 画像 | 5枚以上、**全て具体的**（抽象0） |
| 画像出典 | 全画像にキャプション+出典URL |
| TL;DR | 冒頭5行以内 |
| 出典URL | 10個以上 |
| 体裁 | サンセリフ、16px以上、読みやすい |

### 評価B（不合格 - 再実行）
- 画像が1枚でも抽象的
- WebSearch 15回未満
- 出典URL 10個未満
- 体裁が読みにくい

### 評価C/D（大幅修正必要）
- 画像が不足
- 内容が浅い

## ループ制御

### 最低ループ数: 3回
- 1ループ目: リサーチ & 初稿
- 2ループ目: 画像改善 & 体裁修正
- 3ループ目: 最終チェック & 品質評価

### 完了条件
1. 最低3ループ経過
2. 品質評価A達成
3. 全画像が具体的であることを確認
4. 上記全て満たしたら `SKILL_COMPLETE` 出力

評価Bの場合: 問題点を特定し、次ループで修正
評価C/Dの場合: 大幅に作り直し

## 出力
- output/test/images/ - 取得画像（各10KB以上）
- output/test/content.md - レポート本文
- output/test/report.pdf - 最終PDF（700KB以上目標）
