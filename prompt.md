# 2025デザイントレンド リサーチレポート

## タスク
2025年のグラフィック/UI/UXデザイントレンドについてエキスパートレベルのリサーチレポートをPDF作成。

## 現在のループ状態
ファイル `output/test/loop_state.json` を読んで現在のループ回数を確認。
存在しない場合は1ループ目。ループ完了時に回数を+1して保存。

## 画像ルール（最重要）
### 禁止
- 抽象的イメージ画像（カラーパレット、ワークスペース写真）
- 内容と関係ない装飾画像

### 必須
- 参照記事で実際に使われている図表・スクリーンショット
- 具体的なデータを示すチャート
- 実際のデザイン事例のUI画像
- 画像は本文中の関連箇所に配置

### 画像取得フロー
1. WebFetchで記事を精読
2. 記事内の図表URLを特定
3. Playwrightでスクリーンショット取得
4. 各画像を確認し「具体的か抽象的か」を評価
5. 抽象的なら却下して別を探す
6. **5枚以上の具体的画像**を確保

## 体裁
- サンセリフフォント（Inter, Helvetica）
- 本文16px以上、H1:32px、H2:24px
- 十分な余白

## 品質基準（厳格）
| 項目 | 基準 |
|------|------|
| WebSearch | 15回以上 |
| 深読みソース | 5サイト以上 |
| 画像 | 5枚以上、全て具体的 |
| 出典URL | 10個以上 |
| TL;DR | 冒頭5行 |

## ループ制御
- **最低3ループ必須**
- 1ループ目: リサーチ&初稿
- 2ループ目: 画像改善
- 3ループ目: 最終品質チェック

## 完了条件
1. 最低3ループ経過（loop_state.jsonで確認）
2. 品質評価A達成
3. 全画像が具体的
4. 上記全てOKなら `SKILL_COMPLETE` を最後の行に出力

## 各ループでやること

### ループ1: リサーチ&初稿
- WebSearch 15回以上
- 5サイト以上を深読み
- 画像候補を収集
- 初稿作成
- loop_state.json更新

### ループ2: 画像改善
- 前ループの画像を評価
- 抽象的な画像を差し替え
- 本文改善
- loop_state.json更新

### ループ3: 最終チェック
- 全項目を最終評価
- 問題があれば修正
- 評価Aなら `SKILL_COMPLETE` 出力

output/test/ に出力。
